# Ch3 Page 6 Spec（S14–S15）設計規格

## 目標

製作 `Ch3-Page6-Spec`，用「文件收束型」動畫說明：

- S14：Spec 是將討論內容整理、壓縮後，可長期保存的規格文件。
- S15：Spec 沒有固定格式，但可參考六個重要欄位。

整體風格延續 Page 4、Page 5：白色背景、深色文字、黃色重點、藍色語意元素、暖灰色卡片與幀驅動動畫。

## S14：Spec 定義

### 核心文案

- 標題：`Spec`
- 定義：`把討論整理成可長期保存的規格文件`
- 文件名稱：`feature-spec.md`
- 效益一：`維護與擴充有依據`
- 效益二：`減少重複掃描與 Token 消耗`

### 畫面流程

1. 第一段使用 S11 版型，`Spec` 標題縮放進場。
2. 定義句淡入，黃色強調「長期保存」與「規格文件」，停留後整頁淡出。
3. 第二段讓多張對話卡分散在畫面四周，內容使用短句：
   - 玩家按下空白鍵
   - 角色向上跳躍
   - 落地後才能再次起跳
   - 跳躍高度需要一致
4. 對話卡向中央集中、縮小並堆疊，形成「討論內容」。
5. 集中的卡片轉換成文件卡 `feature-spec.md`。
6. 文件卡穩定後，參考 `Ch2Page2Constraint` 的節奏縮小並移至左側。
7. 右側依序出現兩個效益。
8. 所有內容停留後淡出，銜接 S15。

### 視覺規則

- 對話卡使用暖灰邊框與白底，不模擬真實聊天軟體。
- 四張對話卡以 18 幀為間隔逐張進場，每張使用 30 幀完成 42 px 的向上位移與淡入。
- 收束過程使用位置、縮放、透明度與旋轉，不使用 CSS animation。
- 卡片集中動畫使用 100 幀完成，避免收束過程拖慢敘事節奏。
- 文件卡使用藍色標題列或藍色文件圖示。
- `feature-spec.md` 是畫面中主要視覺焦點。
- 兩個效益使用小型單色 SVG 圖示：
  - 維護／擴充：扳手或分支圖示。
  - Token：掃描線或縮減符號。
- 效益資訊塊沿用既有卡片系統：白底、暖灰框與淡陰影，不使用淡藍色底。

## S15：Spec 結構

### 標題

`Spec 結構`

### 六個欄位

| 欄位 | 中文用途 |
|---|---|
| User Story | 從使用者角度描述功能 |
| Input / Output | 玩家輸入什麼，遊戲回應什麼 |
| Rules | 功能的規則與限制 |
| Non-goals | 這次不實作的範圍 |
| Acceptance Criteria | 怎樣才算完成 |
| Notes | 其他補充與備註 |

### 畫面流程

1. 標題先進場。
2. 六張卡片以兩列三欄排版，作為一個群組同時淡入並微幅上移；可保留 6–10 幀的極短錯開。
3. 目前講解的卡片：
   - 放大約 4%。
   - 邊框切換為黃色。
   - 圖示與欄位名稱切換為黃色。
4. 已講解與尚未講解的卡片維持暖灰色，不完全消失。
5. 六個欄位講解完成後，所有卡片恢復一般狀態。
6. 卡片下方以 S05 相同的藍色純文字樣式顯示：`先看案例，再一起寫 →`。
7. 停留後整頁淡出，銜接 S16 的 Celeste 案例影片。

### 卡片圖示

- User Story：人物。
- Input / Output：雙向箭頭。
- Rules：尺規或清單。
- Non-goals：邊界框或禁止符號。
- Acceptance Criteria：核取方塊。
- Notes：便條紙。

圖示以 React 內嵌 SVG 建立，預設暖灰；高亮時改為黃色。

高亮放大使用 `Easing.bezier(0.16, 1, 0.3, 1)`，形成前段快速、後段緩慢的 ease-out；縮回使用平順的 ease-in-out。
