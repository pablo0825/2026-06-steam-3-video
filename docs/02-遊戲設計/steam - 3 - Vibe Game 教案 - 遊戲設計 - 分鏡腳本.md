# VIBE GAME 教案・第 2 集「遊戲設計」— 分鏡腳本

- **影片型態**：純螢幕 ＋ 旁白配音（SVG 動畫 ＋ 遊戲影片片段 ＋ 螢幕錄影）
- **製作工具**：Remotion（SVG 動畫）＋ 遊戲影片片段 ＋ 螢幕錄影
- **預估片長**：旁白約 2.5～3 分鐘 ＋ 第 8／9 頁實作螢幕錄影 ＋ 第 10 頁片尾名單約 15 秒 ≈ **4～5 分鐘**
- **呈現方式圖例**：`SVG動畫` / `影片素材` / `螢幕錄影` / `文字卡`

> 本分鏡依 02-遊戲設計簡報（`briefing-img/` 共 23 張）與**逐字稿**（`逐字稿.txt`）對齊而成，格式比照第 1 集。
> 旁白採用逐字稿原文（僅修正明顯筆誤）。主要設計決策：
> ① 概念段的遊戲案例（節奏醫生、Celeste）以**真實遊戲影片片段＋旁白**呈現；
> ② 核心玩法先建立定義與觀察任務，進入案例時依序呈現「Celeste 標題 → 遊戲影片 → 三個核心動作」；
> ③ 核心循環的四圈框架維持 SVG，套用範例時搭配遊戲影片；
> ④ 兩段實作以**螢幕錄影**呈現，先以佔位卡暫代。

## 旁白配音稿（純文字，供錄音用；採逐字稿，僅修正明顯筆誤）

1. 嗨，大家好，我是知點的育嘉。這一集我們要跟大家分享，怎麼在遊戲設計階段，導入 AI 來協助我們發想遊戲內容。
2. 這集的重點是，我們會請 AI 來協助我們，產生一份遊戲設計文件，以及 Storyboard。
3. 但在開始做之前，要先跟大家介紹幾個跟遊戲設計文件有關的知識，例如限制設計、核心玩法、核心循環等等。那我們就先從限制設計開始。
4. 限制設計的意思其實很單純，就是在有限的條件下去做設計。這邊可以舉一個大家應該都知道的例子，就是 Game Jam。
5. Game Jam 會要求參賽者，在一天或三天內把遊戲做出來。所以參賽者要在極度有限的時間內，思考有趣的玩法，同時考慮怎麼把遊戲做出來。不過 Game Jam 的限制，是跟活動有關，跟遊戲本體比較沒什麼關係。那我們再介紹一款遊戲，應該有同學看過或玩過。
6. 我們要介紹的遊戲，就是前幾年很紅的節奏醫生。那我們可以先來看一下影片。
7. 節奏醫生可以說是限制設計做得非常明顯的案例。它的玩法是，在節奏到第七拍的時候，按下空白鍵，幫病人按摩心臟。節奏醫生的限制在於，它極簡化玩家的操作，讓玩家只依靠一個按鍵來玩遊戲，並把注意力集中到節拍和時間判斷上。我個人覺得限制設計是不錯的遊戲發想方法，因為有約束的設計，反而會激發設計師的靈感，創造出有趣的體驗，就像節奏醫生這樣的遊戲。
8. 核心玩法，相信大家都有聽過。我們對核心玩法的定義，是指玩家在遊戲中最常重複的動作。這樣講可能有點抽象，所以我們準備了一部影片，大家可以觀察一下，影片中的角色，有哪些動作是最常重複的。
9. （讓大家看影片）
10. 補充一下，這款遊戲叫 Celeste，它是一款以高難度聞名的 2D 平台遊戲。大家經過剛剛的觀察，應該就發現，角色最常重複的動作有三個，分別是跳躍、攀牆、衝刺。這三個動作，就是構成 Celeste 核心玩法的基礎。
11. 接下來是核心循環。由於核心循環比核心玩法還要抽象，所以我們這邊會帶入一個架構，讓大家可以透過這個框架，來理解核心循環是怎麼運作的。
12. 這個框架是由動作、達成、回饋、成長，所組成的循環。我們可以用一個打怪遊戲的例子，來解釋這個框架的運作。
13. 以打怪遊戲為例：動作是遊戲中的主要操作，例如打怪、解謎；達成是透過這些動作完成任務、抵達關卡終點；回饋是通關後獲得的獎勵，例如經驗值、錢；成長是把經驗值拿去升級角色、把錢拿去買更好的裝備，讓角色能挑戰更難的關卡跟更強的怪物。這個循環其實滿常見的，大家可以想一下，自己在玩的打怪遊戲是不是也這樣運作。那我們也準備了另一個案例，讓大家看看不同遊戲的循環，先來看影片。
14. （讓大家看影片）
15. 大家可以發現，Celeste 的循環跟剛剛打怪遊戲的循環不太一樣。Celeste 的循環，是想帶給玩家一種技術上的突破，例如運用自己的技術去突破困難的障礙物，最終抵達關卡終點的體驗；跟打怪遊戲注重角色成長、去挑戰更難關卡的體驗不一樣。這邊想表達的是，循環的設計會影響到玩家的遊戲體驗。所以大家在設計遊戲循環時，可以思考一下，遊戲想帶給玩家什麼樣的體驗。
16. 分鏡圖也有被用在遊戲開發上。在初期開發時，我們可以用分鏡圖，來演出遊戲怎麼玩、關卡是怎麼設計的。
17. 分鏡圖的形式滿多元的，沒有什麼特定的格式。最大的標準，應該就是你的導師、合作夥伴看得懂，就沒什麼問題。
